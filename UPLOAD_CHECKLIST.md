# Upload Checklist

Use this checklist before publishing this project.

## Safe To Publish

- neutral project name
- sample data only
- generic validation logic
- generic duplicate and exception checks
- no internal company names
- no credentials
- no portal-specific automation

## Do Not Publish

- original `Whitestone.zip`
- original `Database.xlsx`
- original credential structure
- original portal names and URLs
- internal runbooks that identify the client workflow

## Suggested Repo Name

- `entity-registry-workflow`
- `company-records-qa-pipeline`
- `record-validation-and-exception-tracking`

## Suggested GitHub Description

Python workflow for validating company records, standardizing fields, flagging exceptions, and producing clean output for downstream reporting.

## Suggested Resume Line

Built a Python-based data QA workflow to validate company records, standardize fields, flag exceptions, and improve reliability of downstream reporting and record management.
